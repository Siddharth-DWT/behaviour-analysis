6. Correct algorithm design

Use this pipeline:

1. Build per-face lip-motion timeline
2. Estimate audio-video offset
3. Shift diarization timestamps
4. Score every Face_N × Speaker_N pair
5. Add segment-level votes
6. Add margin/confidence checks
7. Use Hungarian/bipartite assignment
8. Export only confirmed links
9. Gateway uses only confirmed links for registry fusion
Step A — replace absolute lip score with lip-motion score

Current score is absolute. Add delta:

lip_motion = abs(curr_lip_score - prev_lip_score)

Better:

lip_raw = (
    jaw_open * 0.45
    + mouth_lower_down * 0.25
    + mouth_funnel * 0.15
    + mouth_pucker * 0.10
    + (1.0 - mouth_close) * 0.05
)

lip_motion = abs(lip_raw - prev_lip_raw_for_same_face)

Then normalize per face:

median = np.median(values)
mad = np.median(np.abs(values - median)) + 1e-6
norm_lip = (value - median) / mad

This prevents big/close faces from always winning.

Step B — estimate AV offset first

Before mapping:

global_audio_activity(t) = 1 if anyone is speaking
global_lip_activity(t) = max normalized lip motion across faces

Try offsets:

-1500ms to +1500ms, step 100ms

Pick the offset with highest correlation:

best_offset = argmax(
    mean(global_lip_activity during shifted audio speech)
    - mean(global_lip_activity during shifted silence)
)

Then shift all diarization segments by best_offset.

This is very important because even a 500ms audio/video drift can map the speaking face incorrectly.

Step C — segment-level voting

For every diarized segment:

Speaker_2: 12000ms → 14600ms

Check all faces during that exact segment. The winner is the face with highest lip motion.

Accept the vote only if:

segment duration >= 500ms
face visible ratio >= 0.50
winner score >= threshold
winner - second_best >= margin
not heavy overlap speech

Then:

votes[speaker][face] += segment_duration_ms * confidence

This is stronger than one global average.

Step D — pairwise score

For every (Speaker_N, Face_N):

final_score =
  0.45 * lip_sync_contrast
+ 0.35 * segment_vote_score
+ 0.10 * silence_penalty_inverse
+ 0.10 * temporal_consistency

Reject if:

final_score < 0.30
score_margin < 0.10
matched_speech_ms < 3000
segment_votes < 2
face_visible_ratio < 0.40

Current 0.02 should not be used for identity.

Step E — Hungarian assignment instead of greedy

Current greedy assignment can pick the best local pair and block the correct global pair. Replace it with max-weight bipartite matching.

For a small number of meeting participants, brute force is also okay:

from itertools import permutations

def best_assignment(speakers, faces, score_matrix):
    best = None
    best_score = -1e9

    for face_perm in permutations(faces, min(len(speakers), len(faces))):
        total = 0
        pairs = []
        for speaker, face in zip(speakers, face_perm):
            total += score_matrix.get((speaker, face), -999)
            pairs.append((speaker, face))

        if total > best_score:
            best_score = total
            best = pairs

    return best or []

For production, use scipy.optimize.linear_sum_assignment if available.

Final algorithm

Use two stages:

Stage 1: collect evidence

For every speaker segment:

for segment in diar_segments:
    speaker = segment["speaker"]
    start = segment["start_ms"]
    end = segment["end_ms"]

    for face in faces:
        score = mean_lip_activity(face, start, end)
        visible = face_visible_ratio(face, start, end)

    winner_face = face_with_highest_score
    second_face = second_highest_score

    if winner_score - second_score >= margin:
        votes[speaker][winner_face] += segment_duration * winner_score
Stage 2: lock mapping

After all segments:

for speaker in speakers:
    best_face = highest_vote_face
    second_face = second_highest_vote_face

    if best_score >= threshold and best_score - second_score >= margin:
        lock[speaker] = best_face

Then enforce one-to-one mapping:

Speaker_0 can map only to Face_10
Face_10 can map only to Speaker_0

If two speakers fight for the same face, choose the speaker with stronger evidence and mark the other as unlinked.

Why “mean time” alone is not enough

Mean timestamp overlap is helpful, but alone it can fail because:

audio/video offset may exist
two people may overlap while speaking
listener may move lips/smile silently
face may disappear for a few windows
diarization speaker labels may have short wrong segments
ArcFace may duplicate one person into multiple Face_N tracks

So use mean lip activity inside transcript timestamp windows, but also add:

margin check
minimum duration
minimum votes
one-to-one locking
gateway safety check
The best practical design

Use this rule:

Once Speaker_N ↔ Face_N is confirmed, lock it for the full session.
But before confirmation, keep it as pending.
